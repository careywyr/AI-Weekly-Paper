## Are We Ready For An Agent-Native Memory System?

[我们准备好迎接智能体原生记忆系统了吗？](https://arxiv.org/abs/2606.24775)

大语言模型智能体的记忆系统已演变为复杂的数据管理框架，需要跨多个模块和工作负载进行系统评估，以理解其性能特征与权衡取舍。大语言模型（LLM）智能体的记忆已从简单的检索增强机制迅速演进为支持持久信息存储、检索、更新、整合以及整个智能体执行过程中动态生命周期治理的数据管理系统。尽管经历了这一演进，现有评估仍主要通过端到端任务成功指标（如F1、BLEU）来基准测试智能体记忆，同时将底层系统视为一个整体黑盒。因此，关键的系统级问题——包括运营成本、跨记忆模块的架构权衡，以及动态知识更新下的鲁棒性——仍未得到充分探索。本文从数据管理视角对智能体记忆进行了系统性实验研究。我们提出了一个分析框架，将智能体记忆分解为四个核心模块：记忆表示与存储、提取、检索与路由，以及维护。在此框架下，我们评估了12个代表性记忆系统和两个参考基线，涵盖跨越11个数据集的五个基准工作负载。我们广泛的端到端评估表明，没有单一架构能在所有场景中占据主导地位；相反，有效性在很大程度上取决于记忆结构与工作负载瓶颈的匹配程度。此外，通过细粒度消融研究，我们量化了各模块对表示保真度、检索精度、更新正确性和长期稳定性的个体影响。最后，我们揭示了实际工作负载下的成本-性能权衡，表明局部维护比全局重组更具成本效益。基于这些发现，我们指出了构建真正智能体原生记忆系统的有前景方向。代码已开源在 https://github.com/OpenDataBox/MemoryData。

## PlanBench-XL: Evaluating Long-Horizon Planning of LLM Tool-Use Agents in Large-Scale Tool Ecosystems

[PlanBench-XL：在大规模工具生态系统中评估 LLM 工具使用智能体的长程规划能力](https://arxiv.org/abs/2606.22388)

PlanBench-XL 评估了大语言模型智能体在复杂、工具丰富的环境中，面对有限可见性和动态干扰时的规划与适应能力。LLM 智能体越来越多地运行在大型工具生态系统中，现实世界任务要求它们发现相关工具、推断隐含的子目标，并在长程任务中适应动态环境。然而，现有基准很少评估在检索受限的工具可见性下的规划能力。为填补这一空白，我们提出了 PlanBench-XL，这是一个包含 327 个零售任务、涉及 1665 个工具的交互式基准，用于测试智能体能否迭代检索可用工具、调用它们以揭示中间证据，从而为后续调用最终目标服务。PlanBench-XL 还具备一个可选的阻塞机制，通过缺失、失败或干扰性的工具功能来模拟现实世界中的不可预测性，迫使智能体在运行时检测中断的路径并进行适应。在十个领先的 LLM 上进行的实验表明，大规模工具规划仍然具有挑战性：虽然 GPT-5.4 在无阻塞设置下达到了 51.90% 的准确率，但在最严重的阻塞条件下，其准确率骤降至 11.36%。进一步分析表明，当失败缺乏明确的错误信号，或恢复需要更长的替代工具使用路径时，智能体尤其脆弱。这些结果将 PlanBench-XL 确立为诊断智能体规划失败的测试平台，并凸显了在大型、不完美的工具环境中进行长程任务时，对稳健自适应规划的需求。

## Wan-Streamer v0.1: End-to-end Real-time Interactive Foundation Models

[Wan-Streamer v0.1：端到端实时交互基础模型](https://arxiv.org/abs/2606.25041)

Wan-Streamer 是一个统一的端到端多模态模型，通过因果注意力机制以及对视觉、音频和文本模态的集成处理，实现了实时音视频交互。我们提出了 Wan-Streamer，这是一个原生流式、端到端的交互式基础模型，从头设计用于实时、低延迟、全双工的音视频交互。Wan-Streamer 在单个 Transformer 内无缝地将语言、音频和视频同时建模为输入和输出，其中序列由交错的视觉、音频和文本输入 Token 以及视觉、音频和文本输出 Token 组成，并通过块因果注意力（block-causal attention）进行协调以实现增量流式处理。与依赖独立 VAD、ASR、语言、TTS、音频驱动动画或视频生成模块的级联交互系统不同，Wan-Streamer 不依赖外部语言、语音、虚拟形象或视频生成模块：感知、推理、生成、响应时机、话轮管理以及跨模态同步都在一个统一模型内联合学习，从而减少了流水线延迟和错误累积。为了支持自然的音视频响应能力，我们围绕流式能力重新设计了整个技术栈，包括因果编码器、因果解码器、块因果注意力以及低延迟多模态 Token 调度，使得在 25 fps 下流式单元可短至 160 毫秒。Wan-Streamer 实现了约 200 毫秒的模型端响应延迟，结合 350 毫秒的双向网络延迟后，总交互延迟约为 550 毫秒，支持亚秒级双工音视频通信。这些成果使 Wan-Streamer 成为面向低延迟流式交互的统一、端到端、多模态交互基础模型。

## OpenRath: Session-Centered Runtime State for Agent Systems

[OpenRath：以会话为中心的多智能体系统运行时状态](https://arxiv.org/abs/2606.19409)

OpenRath 为多智能体系统引入了一种类似 PyTorch 的编程模型，以 Session（会话）作为核心运行时抽象，支持显式的分支（fork）、合并（merge）和重放（replay）操作，同时记录完整的执行状态。现代智能体系统常常面临运行时状态碎片化的问题：对话记录、工具效果、记忆事件、工作空间布局、分支来源和重放证据被分别存储，难以检查或复现。OpenRath 通过为多智能体、多会话系统提供类似 PyTorch 的编程模型来解决这一问题。这里的类比指的是核心一等运行时抽象的角色，而非张量计算。其核心抽象是 Session，即在智能体和工作流之间传递的运行时值。Session 是可分支、可检查、可重放、可感知后端且可组合的。它记录对话片段、沙箱位置、谱系元数据、Token 用量、待处理工作和工具证据，同时定义记忆交互进入运行时记录的位置。由于这些状态由程序执行中使用的同一个值携带，分支、合并和重放成为显式的运行时操作，而非从外部轨迹重建的状态。OpenRath 进一步定义了 Sandbox（沙箱）、Tool（工具）、Agent（智能体）、Memory（记忆）、Workflow（工作流）和 Selector（选择器），其中 Selector 将控制流转化为运行时路由的决策。本报告介绍了编程模型、架构、审计里程碑和证据协议。其主张仅限于可控的运行时属性，而广泛的定量比较、实时提供商质量、可选后端可用性和记忆质量留待后续评估。核心论点是：Session 为智能体系统提供了一种可审计组合的一等运行时值。

## Grouped Query Experts: Mixture-of-Experts on GQA Self-Attention

[分组查询专家：GQA 自注意力上的混合专家模型](https://arxiv.org/abs/2606.20945)

分组查询专家（GQE）通过根据 Token 内容选择性激活查询头来提升 Transformer 效率，同时保留分组查询注意力（GQA）的键值缓存优势。自注意力是 Transformer 性能的核心，在长上下文场景下往往成为最昂贵的部分，因为其成对 Token 交互的计算量随序列长度呈二次方增长。标准密集注意力对所有 Token 使用相同的注意力头集合，无论 Token 难度或信息含量如何。这种统一激活会浪费计算资源，尤其在序列变长、注意力成本快速上升时。我们提出分组查询专家（GQE），即在分组查询注意力（GQA）之上构建的混合专家层。在每个 GQA 组内，路由器为每个 Token 选择 k 个查询头专家，而所有键值（KV）头保持密集且不变。因此，GQE 保留了 GQA 的 KV 缓存优势，仅减少活跃查询头的计算量。在 250M 参数规模、固定 30B Token 预算下，GQE 在下游准确率上与全激活 GQA 基线持平，同时每个 Token 仅激活一半的查询头。

## DomainShuttle: Freeform Open Domain Subject-driven Text-to-video Generation

[DomainShuttle: 自由开放域主体驱动文本到视频生成](https://arxiv.org/abs/2606.26058)

## PerceptionDLM: Parallel Region Perception with Multimodal Diffusion Language Models

[PerceptionDLM：基于多模态扩散语言模型的并行区域感知](https://arxiv.org/abs/2606.19534)

PerceptionDLM 通过结构化注意力掩码和高效提示，在多模态扩散语言模型中实现了高效的并行区域感知，在不牺牲描述质量的前提下显著提升推理速度。多模态大语言模型（MLLM）在视觉理解任务中取得了显著进展，但现有模型大多依赖自回归生成，这限制了其在需要描述多个区域的感知任务中的效率。本文提出 PerceptionDLM，一种专为高效并行区域感知优化的多模态扩散语言模型。该架构基于 PerceptionDLM-Base（一个在开源扩散 MLLM 中达到 SOTA 性能的强基线模型），充分利用了扩散语言模型（DLM）的并行解码特性。具体而言，我们引入了高效提示和结构化注意力掩码，使模型能够同时感知多个掩码区域，在序列和 Token 级别并行生成区域描述。与现有逐区域处理的方法相比，该设计显著提升了推理效率。为系统评估 DLM 视觉感知能力的并行性，我们通过扩展 DLC-Bench，为每张图像引入多个区域掩码，构建了新的并行详细局部描述基准（ParaDLC-Bench），可联合评估描述质量和推理效率。实验表明，PerceptionDLM 在区域描述任务中保持竞争力，同时在多区域感知任务中实现了显著的加速。我们的结果凸显了多模态扩散语言模型在高效并行视觉感知方面的潜力。据我们所知，这是首次利用扩散语言模型的优势实现并行区域描述与感知。代码、模型和数据集均已开源。

## NatureBench: Can Coding Agents Match the Published SOTA of Nature-Family Papers?

[NatureBench: 编程智能体能否达到《自然》系列论文已发表的 SOTA？](https://arxiv.org/abs/2606.24530)

NatureBench 提出了一个跨学科基准，包含从《自然》出版物中提取的 90 项科学任务，用于评估 AI 编程智能体实现科学发现（而非仅仅复现）的能力。结果表明，当前智能体主要依赖方法论翻译，而非真正的科学创新。我们推出了 NatureBench，这是一个跨学科基准，包含从经同行评审的《自然》系列出版物中提炼出的 90 项任务，旨在评估 AI 编程智能体能否在真实科学问题上超越复现，迈向发现。NatureBench 基于 NatureGym 构建，后者是一个自动化流水线，能够根据源论文为每项任务构建标准化的容器化环境，解决了此前智能体研究基准因环境碎片化而可信度受限的问题。在严格的禁用网络搜索协议下评估了十种前沿智能体配置后，我们发现，在 g>0.1 标准下，最强模型仅在 17.8% 的任务上超越了 SOTA。对方法路径的分析表明，智能体主要通过方法论翻译（将科学任务转化为熟悉的监督预测问题）取得成功，而非通过真正的科学发明。失败的主要原因在于方法选择错误和计算预算不足，而非任务理解错误。我们发布了该基准、NatureGym 流水线以及一个由维护方负责复现的公开排行榜。代码：https://github.com/FrontisAI/NatureBench

## World Action Models: A Survey

[世界动作模型综述](https://arxiv.org/abs/2606.20781)

世界动作模型（World Action Models, WAMs）是一种预测-动作系统，通过生成未来状态来辅助决策，其设计需要在表征丰富度与计算约束之间取得平衡。由 Qwen/Qwen2.5-Coder-32B-Instruct 生成的 WAMs 是具身预测-动作模型，能够为动作提供未来预测。近期 WAMs 重新利用大规模视频生成模型，另一条并行路线则依赖语言或视觉-语言骨干网络，不包含视频生成核心。这种快速扩展模糊了广义世界模型、视频生成模型、基于动作的视频世界模型、视觉-语言-动作策略以及 WAMs 之间的边界。本综述为该领域提供了统一框架。首先厘清这些边界，然后通过两个互补视角组织现有工作：第一个视角考察每种方法需要生成什么——涵盖渲染未来、隐式未来以及无视频生成的动作推理；第二个视角则按预测基底、骨干网络、动作耦合方式和部署模式对每种方法进行分解。这种剖析支持对可交互性、因果性、持久性、物理合理性和泛化性进行统一讨论，并进一步涉及数据、评估和开放挑战。在这些维度上，一个一致的设计模式浮现出来：WAMs 并非简单的带动作头的视频生成器，而是预测-动作方法，其设计选择在表征丰富度与计算、内存、延迟和动作标签成本之间进行权衡。该领域正朝着生成更少未来内容、同时保留控制所需信息的方向发展。综述主页见 https://world-action-models.github.io/。

## Escaping the Self-Confirmation Trap: An Execute-Distill-Verify Paradigm for Agentic Experience Learning

[逃离自我确认陷阱：面向智能体经验学习的执行-蒸馏-验证范式](https://arxiv.org/abs/2606.24428)

EDV 是一个三阶段框架，利用多个异构智能体协同为 LLM 智能体构建可靠经验，通过执行-蒸馏-验证流程防止自我确认错误。经验驱动的自我进化对于大语言模型（LLM）智能体在开放世界交互中持续提升至关重要。然而，现有经验学习方法大多依赖单智能体循环，即同一智能体同时负责执行任务、总结结果和决定记忆内容。这种设置使智能体容易陷入"自我确认陷阱"：错误但自洽的轨迹被误判为成功经验，导致检索和复用过程中错误累积。为解决该问题，我们提出 EDV——一个面向可靠经验学习的执行-蒸馏-验证框架。在执行阶段，多个异构智能体并行探索同一任务空间，生成多样化的候选轨迹。在蒸馏阶段，专门的第三方智能体通过对比分析这些轨迹生成候选经验，减少以执行者为中心的总结偏差。在验证阶段，执行组通过共识机制验证候选经验，只有通过验证的经验才会写入共享或私有记忆。通过解耦三个阶段，EDV 将经验学习从孤立的自我反思转变为协同构建，在记忆插入前过滤错误和噪声内容。我们在三个具有挑战性的长程基准测试——tau2-bench、Mind2Web 和 MMTB 上评估了 EDV。结果表明 EDV 持续优于强基线，验证了可靠经验构建对于稳健的智能体自我进化至关重要。我们的代码已开源在 https://github.com/shidingz/EDV。

## In-Context World Modeling for Robotic Control

[面向机器人控制的上下文世界建模](https://arxiv.org/abs/2606.26025)

ICWM 使机器人策略能够从自主生成的交互中推断系统变量，通过将系统辨识视为上下文自适应问题，无需参数更新即可适应新配置。现代视觉-语言-动作（VLA）模型通常难以泛化到新场景（如改变相机视角或机器人形态），因为它们通常仅基于当前观测和语言指令进行条件化。由于忽略底层系统配置这一变量，这些模型隐式假设了训练中遇到的固定执行环境，导致任何新环境都需要数据密集型的微调。本文提出上下文世界建模（ICWM）框架，将系统辨识视为上下文自适应问题。ICWM 使机器人策略能够从短历史的自生成、任务无关交互中自主推断关键系统变量。与使用示范来指定执行任务的传统上下文学习不同，ICWM 利用上下文窗口来理解系统如何运作。通过在任务执行前处理这些交互，模型隐式捕获当前系统的世界动态，从而无需参数更新即可适应新配置。在仿真和真实机器人平台上的大量实验表明，ICWM 在新相机视角下显著优于标准 VLA 基线。

## KaLM-Reranker-V1: Fast but Not Late Interaction for Compressed Document Reranking

[KaLM-Reranker-V1：快速但非延迟交互的压缩文档重排序](https://arxiv.org/abs/2606.22807)

KaLM-Reranker-V1 是一种快速重排序器，采用编码器-解码器架构，通过 Matryoshka 嵌入池化和交叉注意力机制解耦查询与段落计算，实现高效的相关性建模。随着检索系统规模扩大，高质量重排序变得愈发重要。然而，现有大多数基于编码器或解码器的重排序器都会联合编码查询和段落，导致计算紧密耦合，限制了部署效率和灵活性。我们提出 KaLM-Reranker-V1，一种快速但非延迟交互（FBNL）的重排序器，它在解耦查询与段落计算的同时保留了富有表现力的相关性建模能力。基于编码器-解码器架构，KaLM-Reranker-V1 使用编码器通过 Matryoshka 嵌入池化预编码段落，而解码器则建模系统指令、用户指令和查询意图；随后通过交叉注意力捕获查询上下文与段落表示之间的相关性。这种设计通过解耦的段落编码实现了高效性，同时通过交叉注意力保留了丰富的相关性建模，因此并非延迟交互。我们实例化了三种规模的 KaLM-Reranker-V1：Nano、Small 和 Large，分别具有 0.27B、1B 和 4B 激活参数。在 BEIR、MIRACL 和 LMEB 上的大量实验表明，KaLM-Reranker-V1 在实现优异重排序性能的同时具备卓越效率。在 BEIR 上，KaLM-Reranker-V1 达到了最先进性能，与 Qwen3-Reranker 系列等强大工业模型持平；在 MIRACL 上，尽管未经过多语言数据的广泛训练，KaLM-Reranker-V1 仍展现出出色的重排序性能。此外，在 LMEB 上，重排序模型展现出明显优势，即使是 0.27B 的 Nano 模型也能与 7-12B 的嵌入模型竞争。

## OpenThoughts-Agent: Data Recipes for Agentic Models

[OpenThoughts-Agent: 智能体模型的数据配方](https://arxiv.org/abs/2606.24855)

本文提出了一套用于训练智能体语言模型（agentic language model）的开源数据策展流水线，通过系统性实验和可扩展训练数据展现了卓越性能。智能体语言模型极大拓展了AI的应用场景，但关于如何为通用型智能体策展训练数据的公开知识仍然匮乏。现有的开源项目如SWE-Smith、SERA和Nemotron-Terminal通常针对单一基准，未能解决如何训练能泛化到多种智能体任务的模型。OpenThoughts-Agent（OT-Agent）项目通过完全开源的数据策展流水线填补了这一空白。我们开展了超过100项受控消融实验，系统性地探究流水线的每个阶段，揭示了任务来源和多样性的重要性。随后，我们从流水线中构建了包含10万条样本的训练集，并在该数据集上微调Qwen3-32B，在七个智能体基准上取得了平均44.8%的准确率，比现有最强的开源数据智能体模型（Nemotron-Terminal-32B，40.9%）提升了3.9个百分点。此外，我们的训练数据展现出强大的可扩展特性，在计算量受控的比较中，每个训练集规模下均优于其他开源数据集。我们已在openthoughts.ai公开发布训练集、数据流水线、实验数据和模型，以支持未来智能体模型训练的开源研究。
