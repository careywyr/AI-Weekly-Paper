## Looped World Models

[循环世界模型](https://arxiv.org/abs/2606.18208)

循环世界模型通过共享 Transformer 块实现迭代式潜在状态精炼，在根据预测复杂度自适应调整计算深度的同时，达到了 100 倍的参数效率。当前世界模型面临一个根本性矛盾：忠实的长程模拟需要深层计算，但深层模型部署成本高且容易产生累积误差。我们通过引入循环世界模型（LoopWM）解决了这一问题，这是首个用于世界建模的循环架构。我们的方法通过参数共享的 Transformer 块迭代精炼潜在环境状态，相比传统方法实现了高达 100 倍的参数效率，同时自适应计算能根据每个预测步骤的复杂度自动调整深度。与扩展模型规模和训练数据正交，LoopWM 将迭代潜在深度确立为世界模拟的新缩放轴，这可能极大推动该领域的发展。

## LoopCoder-v2: Only Loop Once for Efficient Test-Time Computation Scaling

[LoopCoder-v2：仅循环一次即可实现高效的测试时计算扩展](https://arxiv.org/abs/2606.18023)

并行循环 Transformer 通过两次循环获得更精细的表示，从而在代码生成任务上取得更优性能；但额外循环会导致收益递减和位置错位成本增加。循环 Transformer 通过重复应用共享模块来扩展隐式计算，但顺序循环会随着循环次数增加而提升延迟和 KV-cache 内存占用。并行循环 Transformer（PLT）通过跨循环位置偏移（CLP）和共享 KV 门控滑动窗口注意力机制缓解了这一成本，使循环次数成为实际可调的设计参数。因此，我们从收益-成本视角研究 PLT 的循环次数选择：额外循环可能优化表示，但 CLP 也会在每个循环边界引入位置错位。为验证这一研究，我们从头训练了 LoopCoder-v2——一个包含不同循环次数的 7B 参数 PLT 代码生成模型家族，在 18T token 上预训练后，进行匹配的指令微调和评估。实验表明，两循环变体在代码生成、代码推理、智能体软件工程和工具使用基准测试中均优于无循环基线，将 SWE-bench Verified 从 43.0 提升至 64.4 分，Multi-SWE 从 14.0 提升至 31.0 分。相比之下，三次及以上循环的变体出现性能倒退，揭示了强烈的非单调循环次数效应。我们的诊断分析显示，第二次循环提供了主要的生产性优化，而后续循环仅产生递减的振荡更新和降低的表示多样性。由于 CLP 引起的位置错位成本基本固定，而优化收益逐渐萎缩，偏移成本逐渐占据主导。这种收益-成本权衡解释了 PLT 在两次循环时达到饱和的现象，并为循环次数选择提供了诊断依据。

## APPO: Agentic Procedural Policy Optimization

[APPO：智能体程序化策略优化](https://arxiv.org/abs/2606.12384)

一种智能体强化学习方法，通过细粒度决策点和程序级优势缩放来优化分支决策与信用分配，从而提升多轮工具使用能力。近期智能体强化学习（Agentic RL）的进展显著提升了大型语言模型智能体的多轮工具使用能力。然而，现有方法大多基于粗粒度的启发式单元（如工具调用边界或固定工作流）进行信用分配，难以识别哪些中间决策影响了最终结果。本文从两个角度研究智能体强化学习：分支位置选择以及分支后的信用分配方式。初步分析表明，有影响力的决策点广泛分布于生成的整个序列中，而非集中在工具调用处；同时，仅凭 Token 熵无法可靠反映其对最终结果的影响。基于这些发现，我们提出智能体程序化策略优化（APPO），将分支与信用分配从粗粒度的交互单元转移到序列中的细粒度决策点。APPO 使用分支分数（Branching Score）选择分支位置，该分数结合了 Token 不确定性与策略引发的后续延续似然增益，从而在过滤虚假高熵位置的同时实现更具针对性的探索。此外，APPO 引入程序级优势缩放（procedure-level advantage scaling），以更好地在分支展开中分配信用。在 13 个基准测试上的实验表明，APPO 在保持高效工具调用和行为可解释性的同时，将强智能体强化学习基线的一致提升近 4 个百分点。

## Learning from the Self-future: On-policy Self-distillation for dLLMs

[学习自未来：面向扩散语言模型的同策略自蒸馏方法](https://arxiv.org/abs/2606.18195)

d-OPSD 提出了一种面向扩散语言模型的新型同策略自蒸馏框架，通过调整自教师构建和监督机制来适配扩散模型的非自回归特性。同策略自蒸馏（OPSD）已被证明能有效用于大语言模型（LLM）的后训练，但其在扩散语言模型（dLLM）上的应用尚属空白。现有 OPSD 方法本质上是自回归中心的——它们通过从左到右的前缀条件注入特权信息，并采用 Token 级散度监督，这种设计与 dLLM 的任意顺序生成存在根本冲突。我们提出 d-OPSD，这是首个专为 dLLM 设计的 OPSD 框架。我们的方法包含两项核心贡献：首先，我们重构了自教师构建方式，使用自生成答案作为后缀条件，使学生模型能够从"自我未来经验"而非特权前缀中学习；其次，我们将监督从 Token 级迁移至步骤级，使训练与 dLLM 的迭代去噪过程对齐。在四个推理基准上的实验表明，d-OPSD 在样本效率上持续优于 RLVR 和 SFT 基线，仅需 RLVR 约 10% 的优化步骤，为 dLLM 后训练开辟了有前景的路径。代码已开源：https://github.com/xingzhejun/d-OPSD

## Memory is Reconstructed, Not Retrieved: Graph Memory for LLM Agents

[记忆是重构的，而非检索：面向 LLM Agent 的图记忆](https://arxiv.org/abs/2606.06036)

MRAgent 将联想记忆图与主动重构相结合，在推理过程中实现动态记忆访问，从而在降低计算成本的同时提升了长程记忆推理能力。尽管近期取得了进展，LLM Agent 在处理长交互历史的推理时仍面临挑战。当前基于记忆增强的 Agent 依赖静态的“先检索后推理”范式，这种僵化的流水线设计使其无法根据推理过程中发现的中间证据动态调整记忆访问。为弥补这一不足，我们提出了 MRAgent 框架，该框架将联想记忆图与主动重构机制相结合。我们将记忆表示为“线索-标签-内容”图，其中联想标签作为语义桥梁，连接细粒度线索与记忆内容。基于该结构，我们的主动重构机制将 LLM 推理直接融入记忆访问过程，使 Agent 能够根据累积证据迭代探索并剪枝检索路径。这确保了记忆检索能动态适应推理上下文，同时避免无约束扩展导致的组合爆炸。在 LoCoMo 和 LongMemEval 基准上的实验表明，该方法相比强基线取得了显著提升（最高达 23%），同时大幅降低了 Token 和运行时间成本，凸显了主动联想重构在长程记忆推理中的有效性。

## DragMesh-2: Physically Plausible Dexterous Hand-Object Interaction with Articulated Objects

[DragMesh-2：面向铰接物体的物理合理灵巧手物交互](https://arxiv.org/abs/2606.15133)

DragMesh-2 通过接触驱动操控实现灵巧的手物交互，其 PICA 机制可在无触觉反馈的情况下增强不同接触负载下的鲁棒性。铰接物体的灵巧交互对家庭、辅助和人形机器人操控至关重要，多指手能提供超越平行夹爪抓取的柔顺接触模式。然而，铰接物体操控不同于静态物体操控：目标部件无法直接驱动，其运动必须通过持续的手柄物理接触才能产生。这使得从以物体为中心的铰接生成过渡到以手驱动的灵巧手物交互变得困难，因为几何轨迹回放或开环执行无法建模移动铰接部件所需的接触动力学。此外，仅在固定动力学下为完成任务而训练的策略会过度拟合标称接触负载，尤其在缺乏触觉或力反馈时，当接触负载变化时性能可能下降。为解决这些挑战，我们提出 DragMesh-2，一个面向铰接物体灵巧交互的接触驱动框架，将铰接交互从以物体为中心的生成扩展到以手驱动的灵巧手物交互，其中铰接运动必须通过物理接触产生。我们进一步提出 PICA，一种物理信息感知的接触感知训练机制，无需触觉或力反馈即可将物理信号注入策略学习，提升接触负载变化下的鲁棒性和任务成功率。最后，我们在多种阻尼条件和铰接物体类别上进行系统评估，研究接触负载变化下的鲁棒性，并提供纯几何灵巧交互资源以支持未来的移动操控和人形手物交互研究。在七个 GAPartNet 物体上，DragMesh-2 在接触负载变化下比对比方法具有更强的鲁棒性，同时在多种阻尼条件下保持高任务成功率。

## Measuring Epistemic Resilience of LLMs Under Misleading Medical Context

[测量大语言模型在误导性医疗语境下的认知韧性](https://arxiv.org/abs/2606.12291)

大语言模型在接触误导性语境时，其医学推理准确率显著下降，这暴露了当前评估方法的一个关键缺陷——未能衡量模型在对抗性条件下的认知韧性。大语言模型（LLM）如今在医学执业考试中已达到专家级分数，这助长了"高分即安全"的假设，而患者正越来越多地使用它们获取健康建议。我们证明这一假设是脆弱的：当在LLM原本回答正确的问题中注入误导性语境时，它们会放弃正确答案。我们将这种在对抗性语境下保持正确判断的能力称为认知韧性，并引入MedMisBench来测量它。MedMisBench包含10,932道医学题目和48,889对误导性语境-选项组合，涵盖医学推理、智能体能力和患者旅程评估。在11种模型配置下，平均准确率从原始问题的71.1%下降到聚焦误导语境下的38.0%，攻击成功率达51.5%。最具破坏性的注入是形式化、规则式的捏造：权威框架下的虚假陈述攻击成功率达69.5%，例外投毒式主张达64.1%。来自7个国家的14名临床专家评审团认定，38.2%的审查案例存在严重潜在危害。MedMisBench暴露了医学场景下LLM评估的结构性盲点：现有基准衡量模型"知道什么"，却不衡量它们在误导性语境下能否保持正确的医学判断。

## Zone of Proximal Policy Optimization: Teacher in Prompts, Not Gradients

[近端策略优化区：提示中的教师，而非梯度](https://arxiv.org/abs/2606.18216)

近端策略优化区（ZPPO）通过使用重新构造的提示来改进知识蒸馏，帮助学生从正确和错误的回答中学习，尤其在较小模型规模下提升性能。知识蒸馏将教师模型的能力迁移到小型学生模型，但在小模型场景下存在脆弱性：强制学生模仿更大教师模型的 logits 会使其过度关注教师最尖锐的模式，从而损害训练语料之外基准族上的泛化能力。强化学习（RL）通过基于学生自身 rollout 进行训练来避免 logits 模仿。然而，在每次 rollout 都失败的问题上——产生零优势并被静默丢弃——将更强教师模型的响应注入策略梯度会破坏同策略假设并导致漂移。受维果茨基最近发展区理论启发，我们提出近端策略优化区（ZPPO），将教师保留在提示中而非策略梯度中。对于困难问题，ZPPO 构造两种重新构造的提示：二元候选包含问题（BCQ）将一条正确教师响应与一条错误学生响应配对，作为学生必须区分的匿名候选；负候选包含问题（NCQ）将学生的错误 rollout 聚合到单个提示中，以揭示其共有的失败模式。提示重放缓冲区循环处理每个困难问题，直到其毕业——学生在该问题上的平均 rollout 准确率达到一半——或在有限容量下被 FIFO 逐出，从而在学生当前最近发展区内放大 BCQ 和 NCQ。在 Qwen3.5 系列上，使用四个学生规模（0.8B-9B）和一个 27B 教师模型，后训练为视觉语言模型并在 31 个基准测试套件（16 个 VLM、10 个 LLM、5 个视频）上评估，ZPPO 优于同策略/异策略蒸馏和 GRPO，且在最小规模上增益最大。

## GameCraft-Bench: Can Agents Build Playable Games End-to-End in a Real Game Engine?

[GameCraft-Bench: 智能体能否在真实游戏引擎中端到端构建可玩游戏？](https://arxiv.org/abs/2606.17861)

端到端游戏生成为编码智能体带来了重大挑战，要求它们根据自然语言描述创建完整的可玩游戏，同时满足引擎接地、工件完整性和交互验证等特定评估标准。游戏生成是编码智能体的新兴应用，要求模型将自然语言规范转化为可玩的交互系统。与传统编码任务不同，游戏生成在游戏引擎内进行，脚本、场景、资源、渲染和运行时交互必须共同产生连贯的游戏体验。我们将端到端游戏生成形式化为这样一个问题：在目标环境中通过可观察的玩家-游戏交互，生成实现规范的完整游戏工件。我们认为，评估这一场景需要三个必要条件：引擎接地、工件完整性和交互验证。我们提出了一种基于交互的评估框架，通过重放演示和基于评分标准的多模态评判来评估可执行的游戏玩法。我们将该框架实例化为 GameCraft-Bench，这是一个包含 15 个游戏家族、140 个 Godot 任务的基准测试。对前沿编码智能体的评估表明，端到端游戏生成仍然极具挑战性：最强智能体仅达到 41.46%，大多数智能体得分低于 40%。进一步分析显示，虽然智能体通常能实现可识别的机制，但在提供内容充足、功能视觉反馈完整且呈现连贯的完整游戏方面仍存在困难。演示、代码和数据请见 https://tongxuluo.github.io/gamecraft-bench-website。

## ACE-Ego-0: Unifying Egocentric Human and Robotic Data for VLA Pretraining

[ACE-Ego-0：统一第一人称人类与机器人数据用于 VLA 预训练](https://arxiv.org/abs/2606.17200)

一种统一的视觉-语言-动作（VLA）预训练框架，通过一种可靠性感知的训练方法，利用包括人类第一人称视频和机器人轨迹在内的异构数据源，提升了具身 AI 任务的性能。VLA 模型受益于大规模、多样化的具身数据，但收集机器人轨迹成本高昂且劳动密集。近期进展表明，大规模第一人称人类视频在预训练中提供了互补的真实世界监督。然而，由于动作空间、具身结构、时间动态和监督质量上的差异，联合训练人类和机器人数据仍然具有挑战性。我们引入了 ACE-EGO-0，一个统一利用异构数据源的 VLA 预训练框架。为了从第一人称人类视频中提取大规模预训练监督，我们构建了一个可扩展的第一人称视频到动作流水线，将原始人类视频转换为机器人格式的伪动作轨迹。为了使这些标签与机器人演示可比，ACE-EGO-0 使用了一种基于相机空间动作、形态条件化和时间对齐动作分块的统一动作表示。为了稳健地利用来自第一人称人类视频的含噪伪动作监督，我们制定了一个可靠性感知的训练目标，并辅以人类辅助损失，将监督集中在可靠信号上。我们在 4.53K 小时的机器人和模拟数据以及 1.48K 小时的伪动作标注的第一人称人类数据上实例化了 ACE-EGO-0。实验表明，在可靠性感知加权下纳入大规模人类监督，能持续提升统一联合预训练和监督微调的性能。ACE-EGO-0 在 RoboCasa GR1 TableTop 和 RoboTwin 2.0 上达到了最先进的性能，并展现出对真实世界双臂操作的强大迁移能力。

## MolmoMotion: Forecasting Point Trajectories in 3D with Language Instruction

[MolmoMotion：基于语言指令的 3D 点轨迹预测](https://arxiv.org/abs/2606.18558)

3D 点运动预测模型能够根据视觉历史记录和语言目标预测物体轨迹，在基准测试中表现出色，并能有效迁移到机器人操作和视频生成任务中。运动预测是视觉智能的核心：智能体必须预测物体如何运动，才能规划行动、推理物理交互并合成逼真的未来场景。我们认为，世界坐标系中的 3D 点提供了一种通用表示，这种表示与类别无关、视角稳定、紧凑且可直接用于下游任务。我们形式化了目标条件化的 3D 点运动预测任务：给定一段简短的视觉历史记录、目标物体上的一组 3D 查询点以及预期目标的语言描述，模型预测每个点的未来 3D 轨迹。我们引入了一套完整的技术栈来大规模研究该任务：(1) MolmoMotion-1M 是一个大型语料库，包含从 116 万个无约束视频中标注的、带有动作描述和物体锚定的 3D 点轨迹；(2) PointMotionBench 是一个人工验证的基准测试，涵盖 111 个物体类别和 61 种运动类型；(3) MolmoMotion 是一个通用运动预测模型，支持自回归坐标预测和基于流匹配的轨迹生成。MolmoMotion 能根据不同的语言指令准确预测多种运动模式，并在 PointMotionBench 上显著优于现有的运动预测基线。最后，我们展示了学习到的 3D 运动先验能很好地迁移到下游应用：它提高了机器人操作的训练效率和泛化能力，其预测的轨迹为生成模型提供了有效的运动指导，从而合成具有更逼真物体运动的视频。

## Beyond the Current Observation: Evaluating Multimodal Large Language Models in Controllable Non-Markov Games

[超越当前观测：在可控非马尔可夫游戏中评估多模态大语言模型](https://arxiv.org/abs/2606.19338)

一项名为 RNG-Bench 的新基准套件被提出，用于评估多模态基础模型重建过去观测并据此在多步交互中做出决策的能力。该套件包含两个具有可控难度参数的游戏，并引入记忆差距指标来区分遗忘与决策失误。将多模态基础模型部署为闭环策略时，越来越需要根据不再可见的观测来调整动作。然而，现有基准要么暴露完整状态，将隐藏状态重建与其他智能体技能混为一谈，要么仅在回合结束后测试回忆能力。我们提出 RNG-Bench（重建性非马尔可夫游戏），这是一个旨在隔离基础模型重建过去观测并在多步交互中据此行动能力的基准套件。RNG-Bench 包含两个互补游戏：配对记忆游戏，其中特定位置短暂揭示的卡片身份必须在之后被回忆；以及 3D 迷宫游戏，其中自我中心视角必须整合到空间地图中。两个游戏都在统一框架下进行评估，包含三个可控难度维度：网格大小、视觉模式和观测模态。该基准进一步引入头对头对决协议以控制实例级方差，以及一个记忆差距指标来区分遗忘与不良动作选择。最难的配置要求每回合约 128K 令牌和 350 张图像输入的上下文，前沿多模态大语言模型远未达到饱和。记忆差距分析表明，大多数残留错误源于对早期观测的遗忘，而非次优决策。最后，在最优策略轨迹和过滤后的模型演示上微调 Qwen3.5-9B，可提升其在 RNG-Bench 上的性能，并迁移到现有基准，同时不降低通用多模态能力。

## HarnessX: A Composable, Adaptive, and Evolvable Agent Harness Foundry

[HarnessX：可组合、自适应、可演化的智能体框架工厂](https://arxiv.org/abs/2606.14249)

HarnessX 通过组合式原语、轨迹驱动演化和反馈循环，实现了自适应且可演化的 AI Agent 运行时接口，同时改进了框架设计与模型训练。AI Agent 的性能关键取决于运行时框架（runtime harness），包括提示词、工具、记忆和控制流，这些组件决定了模型如何观察、推理和行动。然而，当前的框架大多依赖手工构建且静态不变：每个新模型或新任务仍需定制化脚手架，执行过程中产生的丰富轨迹也极少被系统性地用于改进。我们提出 HarnessX，一个可组合、自适应、可演化的 Agent 框架工厂。HarnessX 通过替换代数（substitution algebra）组装类型化框架原语，借助 AEGIS（一种基于符号适应与强化学习操作镜像的轨迹驱动多 Agent 演化引擎）进行自适应调整，并通过将轨迹转化为框架更新和模型训练信号来闭合框架-模型循环。在五个基准测试（ALFWorld、GAIA、WebShop、tau^3-Bench 和 SWE-bench Verified）上，HarnessX 平均提升 +14.5%（最高达 +44.0%），其中基线越低提升幅度越大。这些结果表明，Agent 的进步不必仅依赖模型规模扩展：从执行反馈中组合和演化运行时接口是一个可行且互补的杠杆。完整代码将在未来版本中开源。

## Orchestra-o1: Omnimodal Agent Orchestration

[Orchestra-o1：全模态智能体编排框架](https://arxiv.org/abs/2606.13707)

本文提出一种全模态智能体编排框架，通过统一任务分解和专用子智能体执行，实现跨多种模态的高效协作，在复杂多模态基准测试中取得卓越性能。近期智能体集群的成功将基于大语言模型（LLM）的智能体从单智能体工作流转向多智能体系统，凸显了智能体编排在任务分解与协作中的重要性。然而，现有编排框架局限于少数模态，难以泛化到异构模态共存互动的复杂场景。这种局限在全模态场景中尤为突出——任务需要统一理解并协调文本、图像、音频、视频等多样化输入。为此，我们提出 Orchestra-o1，一种支持跨模态高效智能体协作的全模态编排框架。Orchestra-o1 引入统一编排机制，实现模态感知的任务分解、在线子智能体特化以及并行子任务执行。这种可扩展设计使智能体系统能够有效处理涉及异构信息源的复杂现实任务，在 OmniGAIA 基准上以 10.3% 的准确率优势超越第二名方法。此外，我们提出决策对齐组相对策略优化（DA-GRPO），一种用于训练 Orchestra-o1-8B 的高效智能体强化学习方法，该方法在所有现有开源全模态智能体中达到最优性能。
