## On the Scaling of PEFT: Towards Million Personal Models of Trillion Parameters

[论 PEFT 的规模化：迈向万亿参数的百万级个人模型](https://arxiv.org/abs/2606.02437)

参数高效微调（PEFT）可以作为持久化个人模型的紧凑基础，通过让小型可训练适配器在强大基础模型之上存储实例特定的行为。参数高效微调通常被视为全参数微调的廉价替代方案。我们研究了一个更广泛的角色：小型可训练适配器作为强大共享基础模型之上的持久化本地状态。在这种框架下，基础模型提供共享能力，而适配器承载实例特定的行为，如偏好、技能、工具习惯和类似记忆的更新。我们围绕三个规模化轴心组织问题：向上规模化（Scale Up），更强的共享先验使小型本地更新更有用；向下规模化（Scale Down），研究适配器可以做到多小同时保持可靠性；向外规模化（Scale Out），许多持久化的适配实例共存。MinT 提供了一个管理适配器身份、版本、来源、评估和服务驻留的基础设施示例。综合来看，结果表明 PEFT 可以成为持久化个人模型的紧凑基础，而不仅仅是全参数微调的预算替代方案。

## Crafter: A Multi-Agent Harness for Editable Scientific Figure Generation from Diverse Inputs

[Crafter: 面向多样化输入的可编辑科学图表生成多智能体框架](https://arxiv.org/abs/2605.30611)

自动生成科学图表的系统在处理多种图表类型和条件时面临局限，这促使研究者开发能够泛化到不同输入场景并生成可编辑输出格式的多智能体框架。科学图表是传达复杂研究思想最有效的手段之一，但制作达到发表质量的插图仍是论文准备中最耗时的环节。现有自动化系统各自针对纯文本输入下的单一图表类型，未能覆盖研究者实际使用的多样类型和条件；其光栅输出也无法进行局部修改。由于科学图表是由离散语义组件构成的结构化组合，生成器在此类布局上产生的局部错误需要的不是更强的基座模型，而是一个编排框架。我们通过两个互补系统实例化该框架：Crafter——一个无需架构变更即可泛化到多种图表类型和输入条件的多智能体图表生成框架；以及CraftEditor——应用相同模式将光栅输出转换为可编辑SVG。此外，我们引入CraftBench基准测试，涵盖三种图表类型和四种输入条件，并配有人工质量标注。实验表明，Crafter在PaperBanana-Bench和CraftBench上显著优于独立生成器和智能体基线，消融实验证实了每个组件的独立贡献；CraftEditor能忠实地将输出转换为可编辑SVG，超越所有基线。代码和基准测试已开源：https://github.com/HaozheZhao/Crafter

## Domino: Decoupling Causal Modeling from Autoregressive Drafting in Speculative Decoding

[Domino：在推测解码中将因果建模与自回归草稿解耦](https://arxiv.org/abs/2605.29707)

Domino 是一种推测解码框架，通过并行主干网络和轻量级因果精化头，将因果依赖建模与自回归草稿生成解耦，从而提升 LLM 推理速度，在端到端执行和吞吐量方面均实现显著加速。推测解码通过草拟多个 Token 并与目标模型并行验证来加速 LLM 推理。然而，其实际加速效果受限于草稿质量与草稿成本之间的权衡：自回归草稿器能建模草稿 Token 间的因果依赖关系，但引入顺序开销；并行草稿器虽降低草稿成本，却削弱了块内依赖建模。本文提出 Domino 框架，将因果依赖建模与昂贵的自回归草稿执行解耦。Domino 首先使用并行草稿主干为整个块生成初步草稿分布，然后通过轻量级 Domino 头利用前缀相关的因果信息对其进行精化。为稳定教师强制因果编码，我们进一步引入基于锚点的训练课程，先强化并行主干，再逐步将优化目标转向因果修正后的最终分布。在 Qwen3 模型上的实验表明，Domino 在 Transformers 后端下实现高达 5.49 倍的端到端加速，在 SGLang 服务下实现高达 5.8 倍的吞吐量加速。

## COLLEAGUE.SKILL: Automated AI Skill Generation via Expert Knowledge Distillation

[COLLEAGUE.SKILL：通过专家知识蒸馏实现自动化 AI 技能生成](https://arxiv.org/abs/2605.31264)

基于人的 AI 技能可从异构痕迹中自动蒸馏为可检查、可修正的包，同时捕获能力与行为模式。由 Qwen/Qwen2.5-Coder-32B-Instruct LLM 生成的智能体不仅需要完成孤立任务，还需承载人类专业知识、判断力和交互风格的有限表征。构建此类基于人的智能体仍然困难，因为与个人或角色相关的可操作知识通常嵌入在异构痕迹中，而非以清晰指令形式存在。现有记忆与人格系统能捕获这些证据的片段，技能框架则提供可移植的封装格式；然而，目前缺乏将痕迹蒸馏为可检查、可修正且智能体可用的技能的端到端工作流。我们提出一种自动化的痕迹到技能蒸馏系统，通过专家知识蒸馏生成基于人的 AI 技能。给定目标人员或角色的材料，COLLEAGUE.SKILL 生成一个带版本号的技能包，包含两个协调轨道：能力轨道（涵盖实践、心智模型和决策启发式）与受限行为轨道（涵盖沟通风格、交互规则和修正历史）。该包可被检查、调用、通过自然语言反馈更新、回滚、跨智能体主机安装，并可选择性地为受控分发做准备。我们描述了开源系统中实现的工件契约、生成工作流、修正生命周期、部署面以及领域预设。截至撰写时，公共仓库拥有约 18.5k GitHub 星标；技能画廊列出了来自 165 位贡献者的 215 项技能，各技能卡累计超过 10 万星标。该系统展示了如何将基于人的技能表示为可移植、可修正的包，而非不透明的提示或隐藏记忆。

## GrepSeek: Training Search Agents for Direct Corpus Interaction

[GrepSeek：训练搜索智能体实现直接语料库交互](https://arxiv.org/abs/2605.29307)

GrepSeek 通过使用 shell 命令进行直接语料库交互，并采用冷启动数据集与组相对策略优化相结合的两阶段方法，实现了高效的搜索智能体训练。由 Qwen/Qwen2.5-Coder-32B-Instruct 大语言模型（LLM）生成的搜索智能体，通过多轮推理和信息检索，在知识密集型语言任务中展现出巨大潜力。现有系统大多使用检索器获取信息，该检索器接收关键词或自然语言查询，并利用预计算文档表示的索引返回排序后的文档列表。本研究探索了一种互补视角：搜索智能体将语料库本身视为搜索环境，通过执行可执行的 shell 命令来寻找证据。我们提出了 GrepSeek，一种优化的直接语料库交互（DCI）搜索智能体，它训练一个紧凑的搜索智能体从大型文本语料库中查找、筛选和组合证据。为了解决直接在大规模语料库上使用强化学习进行行为学习的不稳定性，我们提出了一种两阶段训练流程。首先，我们使用答案感知的 Tutor 和答案盲的 Planner 构建冷启动数据集，生成经过验证且因果有据的搜索轨迹。其次，我们使用组相对策略优化（GRPO）优化初始策略，使智能体能够通过与语料库的直接交互来改进其面向任务的搜索行为。为了使 DCI 在大规模场景下实用化，我们进一步使用语义保持的分片并行执行引擎，将基于 shell 的检索速度提升高达 7.6 倍，同时保持与 shell 命令顺序执行的字节级精确等价。在七个开放域问答基准上的实验表明，GrepSeek 在整体 token 级 F1 和精确匹配指标上取得了最强性能。我们的分析还揭示了纯词汇交互在表面形式变化较大的查询上的局限性，表明 DCI 是一种实用且具有竞争力的搜索智能体方法，可以在现实世界中补充现有的检索范式。

## Audio Interaction Model

[音频交互模型](https://arxiv.org/abs/2606.05121)

本文开发了一种统一的流式音频模型，通过端到端框架将离线任务执行与实时音频指令跟随相结合，支持多种音频交互能力。音频本质上是一种交互式模态，但当前的大型音频语言模型（LALM）均为离线模型，而流式音频模型各自仅处理单一任务，如流式 ASR 或语音聊天。现在是将它们统一为一个在线 LALM 的时候了：这种模型通过始终开启的“感知-决策-响应”循环，实时监听声音、环境和指令，并即时做出反应。我们将这一机制形式化为音频交互模型，并通过 Audio-Interaction 实现——这是一个统一的流式模型，在保留离线任务执行能力的同时，增加了在线通用音频指令跟随功能，涵盖从对话到完整语音聊天的场景，并能根据流式语义决定何时响应。为实现这一目标，我们提出了 SoundFlow 框架，该框架通过流式原生数据构建、理解感知训练以及用于稳定实时交互的异步低延迟推理，端到端地实例化了“感知-决策-响应”循环。我们还构建了 StreamAudio-2M，一个包含 260 万条数据的流式语料库，覆盖 7 项基础能力和 28 个子任务，以及用于评估主动音频干预的 Proactive-Sound-Bench。在 8 个基准测试中，Audio-Interaction 在主流音频任务上保持了有竞争力的性能，同时解锁了离线 LALM 无法实现的能力，包括实时 ASR、流式音频指令跟随和主动帮助。

## Cosmos 3: Omnimodal World Models for Physical AI

Cosmos 3 是一种全模态世界模型，通过统一的混合 Transformer 架构处理和生成多种数据类型，在各类理解与生成任务中达到 SOTA 性能。我们提出 Cosmos 3 系列全模态世界模型，旨在统一的混合 Transformer 架构中联合处理与生成语言、图像、视频、音频和动作序列。通过支持高度灵活的输入输出配置，Cosmos 3 无缝统一了物理 AI 的关键模态——将视觉语言模型、视频生成器、世界模拟器和世界动作模型有效整合到单一框架中。评估表明，Cosmos 3 在多种理解与生成任务上树立了新的 SOTA，证明了全模态世界模型可作为具身智能体的可扩展通用骨干网络。在技术报告撰写时，我们的后训练 Cosmos 3 模型被 Artificial Analysis 评为最佳开源文生图和图生视频模型，并被 RoboArena 评为最佳策略模型。为加速物理 AI 的开放研究与部署，我们在 Linux 基金会的 OpenMDW-1.1 许可下开放了代码、模型检查点、精选合成数据集和评估基准。项目网站见 https://research.nvidia.com/labs/cosmos-lab/cosmos3 。

## OCC-RAG: Optimal Cognitive Core for Faithful Question Answering

[OCC-RAG：面向忠实问答的最优认知核心](https://arxiv.org/abs/2606.00683)

紧凑型任务专用语言模型通过新颖的训练流程和结构化推理轨迹，在多跳推理和忠实性方面展现出优于大型通用模型的性能。语言模型的最新进展一直以规模为特征，每一代模型都将更多世界知识吸收进其参数中。然而，许多实际应用更受益于稳健推理而非广泛的参数化知识。在此背景下，任务专用的小型语言模型（SLM）提供了一种原则性的设计选择。我们提出最优认知核心（OCC），这是一系列基于此前提构建的SLM。作为OCC的变体，我们推出了OCC-RAG，专为基于给定上下文的忠实问答（QA）优化。该任务与OCC设计方法直接对齐，需要在提供的段落上进行多跳推理，同时忽略记忆中的知识。为训练OCC-RAG，我们实现了一种新颖的流水线，用于大规模合成多上下文、多跳QA数据，生成了一个包含超过三百万样本的语料库，专注于多跳推理、严格上下文忠实性和校准的拒绝回答。我们发布了OCC-RAG-0.6B和OCC-RAG-1.7B，两者均在该语料库上进行了中期训练。这些模型生成带有源引用的结构化推理轨迹，这些引用基于上下文中的逐字引用。通过OCC-RAG，我们证明了紧凑型任务专用SLM在多跳推理（HotpotQA、MuSiQue、TAT-QA）、忠实性（ConFiQA）和拒绝回答（MuSiQue-Un）基准测试中，能够匹配或超越规模大2至6倍的通用模型。

## Code2LoRA: Hypernetwork-Generated Adapters for Code Language Models under Software Evolution

[Code2LoRA：面向软件演化的超网络生成适配器，用于代码语言模型](https://arxiv.org/abs/2606.06492)

Code2LoRA 是一个超网络框架，可为代码语言模型生成仓库特定的 LoRA 适配器，通过高效的参数微调同时支持静态和演化的代码库。代码语言模型需要仓库级别的上下文来解决导入、API 和项目约定问题。现有方法通过长输入（通过 RAG 或依赖分析检索）或逐仓库微调和 LoRA 注入这些知识——这在仓库规模下成本高昂，且对演化的代码库脆弱。我们提出 Code2LoRA，一个生成仓库特定 LoRA 适配器的超网络框架，以零推理时 Token 开销有效注入仓库知识。Code2LoRA 支持两种使用场景：Code2LoRA-Static 将单个仓库快照转换为适配器，适用于理解稳定代码库；而 Code2LoRA-Evo 维护一个由 GRU 隐藏状态支持的适配器，该状态按代码差异更新，适用于演化代码库的活跃开发。为评估 Code2LoRA 与参数高效微调基线，我们构建了 RepoPeftBench，一个包含 604 个 Python 仓库的基准，包含两个赛道：静态赛道有 40K 训练和 12K 测试断言补全任务，演化赛道有 215K 提交派生训练和 87K 提交派生测试任务。在静态赛道上，Code2LoRA-Static 实现了 63.8% 跨仓库和 66.2% 仓库内精确匹配，与逐仓库 LoRA 上界持平；在演化赛道上，Code2LoRA-Evo 实现了 60.3% 跨仓库精确匹配（比单个共享 LoRA 高 5.2 个百分点）。Code2LoRA 代码见 https://anonymous.4open.science/r/code2lora-6857；模型检查点和 RepoPeftBench 数据集见 https://huggingface.co/code2lora。

## Trust-Region Behavior Blending for On-Policy Distillation

[面向在线策略蒸馏的信任域行为混合方法](https://arxiv.org/abs/2605.31159)

信任域行为混合（Trust-Region Behavior Blending）通过在线策略蒸馏的预热阶段，在 KL 信任域内用类似教师的行为替换早期低质量的学生轨迹，从而提升蒸馏效果。在线策略蒸馏（On-policy Distillation, OPD）在训练学生模型时，使用从其自身策略采样的前缀，同时匹配更强的教师模型。这解决了离线蒸馏中的前缀不匹配问题，但早期学生轨迹仍然可能质量较差，导致教师监督作用于薄弱或低质量的前缀。我们提出信任域行为混合（TRB），这是一种预热方法，在预热阶段用学生中心 KL 信任域内最接近教师的行为策略替换早期轨迹策略，同时保持每个前缀的反向 KL OPD 损失不变。KL 预算逐渐退火至零，因此预热后训练恢复为纯学生轨迹。在两个数学推理蒸馏设置中，TRB 在对比方法中取得了最强的平均性能。

## A Matter of TASTE: Improving Coverage and Difficulty of Agent Benchmarks

[A Matter of TASTE: 提升智能体基准测试的覆盖度与难度](https://arxiv.org/abs/2605.28556)

自动化基准生成方法通过自适应对比 n-gram 建模与迭代难度优化，演化工具序列，从而创建覆盖更广工具使用范围的高难度任务。由 Qwen/Qwen2.5-Coder-32B-Instruct 生成。随着智能体能力的提升，现有基准（如 τ^2-Bench）正逐渐趋于饱和。然而，构建新的基准任务仍然复杂、昂贵且劳动密集。此外，标准方法——先用自然语言编写场景，再映射到工具序列——仅能捕获智能体实际使用的工具模式中的一小部分子集。本文通过反转任务构建流程来解决这些问题。我们提出 TASTE：基于工具序列演化的任务合成方法，这是一种自动生成高难度、覆盖更广工具使用范围任务的方法。TASTE 利用基于 LLM 判断的有效性信号训练的自适应对比 n-gram 模型，从而能够采样覆盖大量工具组合的有效工具序列。随后，TASTE 通过聚类从序列池中选取代表性序列，将其实例化为完整的基准任务，并通过迭代难度演化进行优化。利用 TASTE，我们构建了 τ^c-Bench，这是 τ^2-Bench 三个领域的具有挑战性的扩展。我们评估了 11 个智能体/用户 LLM 组合，发现在 τ^2-Bench 上几乎饱和的模型在我们的任务上性能大幅下降（例如，Gemini-3-Flash 从 0.82-0.94 降至 0.28-0.61）。除了增加难度外，我们生成的任务使智能体必须执行的独特工具组合数量翻了一番以上。我们的结果表明，现有基准上的高分往往反映的是饱和而非稳健的任务解决能力。通过自动化生成高难度、高覆盖度的基准，TASTE 能够实现对未来智能体的持续、可扩展评估。

## Masking Stale Observations Helps Search Agents -- Until It Doesn't: A Regime Map and Its Mechanism

[遮蔽陈旧观测有助于搜索智能体——直到适得其反：一个机制图谱及其机理](https://arxiv.org/abs/2606.00408)

长程搜索智能体中的观测遮蔽效果呈现非对称倒U型模式，其准确率提升幅度取决于检索器能力与模型容量的交互作用。长程搜索智能体在多次工具调用中积累大量检索内容，使得上下文预算效率愈发重要。一种最小干预手段是在轨迹推进过程中遮蔽上下文中的陈旧观测，但尚不清楚这种上下文管理方式何时有效及其原因。我们通过系统扫描不同规模智能体主干（4B至284B参数）与三种检索器，在离线及在线网页搜索基准上研究观测遮蔽。研究发现，当以无上下文管理时的模型准确率为横轴时，遮蔽带来的准确率增益呈现非对称倒U型：弱检索器下出现平台期，强检索器配合中等容量模型时达到峰值，模型饱和时则急剧下降。这种模式反映的是检索器召回率与模型隐式过滤能力的交互作用，而非单一因素主导。从机制上看，遮蔽实现了token与轮次的权衡：它移除了模型已基本停止关注的观测，以及智能体极少重新打开的页面。新增轮次在将失败转化为成功时发挥作用，但当遮蔽移除了模型本可使用的证据时则适得其反。因此我们将上下文管理重新定义为依赖机制状态的干预手段，并为分析智能体深度搜索中的上下文使用提供整体视角。我们在此发布研究框架与轨迹数据（https://github.com/i-DeepSearch/observation-masking）以支持后续研究。

## Representation Forcing for Bottleneck-Free Unified Multimodal Models

[表示强制：无瓶颈统一多模态模型](https://arxiv.org/abs/2605.31604)

表示强制（Representation Forcing）使统一多模态模型能够端到端地执行感知和生成任务，无需依赖外部潜在空间，在图像生成上达到当前最优性能，同时提升理解能力。统一多模态模型（UMM）旨在用单一模型处理感知与生成。然而现有 UMM 仍依赖冻结且单独预训练的 VAE 进行图像生成，造成结构性瓶颈。直接移除 VAE 会引入质量差距，因为模型必须从原始像素同时学习高层结构和低层细节。本文提出表示强制（RF），通过将表示预测变为模型原生能力来弥合这一差距。具体而言，RF 强制解码器在像素之前自回归地预测视觉表示作为中间 Token；这些 Token 随后保留在上下文中，在同一骨干网络内引导像素扩散。通过将表示从感知输出转变为生成目标，RF 消除了对外部生成潜在空间的需求。我们发现 RF 对理解和生成均有裨益。在图像生成上，采用 RF 的像素空间模型匹配了基于 VAE 的最优统一模型。在图像理解上，像素空间 RF 通常优于其 VAE 变体。这些结果共同为迈向端到端、无瓶颈的 UMM 提供了有效一步。

## SwanVoice: Expressive Long-Form Zero-Shot Speech Synthesis for Both Monologue and Dialogue

[SwanVoice：面向独白与对话的表现力长文本零样本语音合成](https://arxiv.org/abs/2605.30993)

本文提出名为 SwanVoice 的零样本语音合成系统，通过结合 VAE、流匹配 DiT 和扩散后训练技术，解决了富有表现力的长文本多说话人对话合成难题。零样本语音合成在单说话人场景已有显著提升，但表现力丰富的长文本多说话人对话仍具挑战。常见做法是使用独白 TTS 模型逐轮合成再拼接，这增加了推理成本，且常破坏跨轮次的声学一致性、对话连贯性和情感连续性。近期对话 TTS 系统开始应对此场景，但仍难以同时保持表现力连贯性、可控说话人切换和独白质量。我们提出 SwanData-Speech 和 SwanVoice：SwanData-Speech 从野外音频构建独白和对话语料库，使用 Swan 强制对齐器实现停顿感知的词级对齐，并用 RobustMegaTTS3 处理发音困难案例。基于这些数据，SwanVoice 是支持 1-4 说话人的零样本 TTS 模型，结合 25 Hz VAE、带停顿感知符号和拼音替换的原始文本条件，以及带说话人-轮次条件的流匹配 DiT。训练从独白语音开始，经过混合和真实对话数据，再使用带音素级和说话人相似度奖励的 DiffusionNFT 后训练。在 SwanBench-Speech 上，SwanVoice 在独白和对话场景中均获得比所有评估的开源基线更高的丰富度和层次感得分，但内容准确性仍是主要局限。音频演示见 https://swanaigc.github.io//#swanvoice。

## KVarN: Variance-Normalized KV-Cache Quantization Mitigates Error Accumulation in Reasoning Tasks

[KVarN：方差归一化 KV-Cache 量化缓解推理任务中的误差累积](https://arxiv.org/abs/2606.03458)

KVarN 是一种免校准的 KV-Cache 量化器，通过 Hadamard 旋转和双尺度方差归一化，减少大语言模型自回归解码过程中的误差累积。测试时扩展（Test-time scaling）是提升大语言模型推理能力的有效方法，但在长序列解码时，KV-Cache 的增长会导致内存瓶颈。KV-Cache 量化可以改善这一问题，但现有方法仅在预填充（prefill）类设置下评估，而自回归解码中误差行为有所不同。我们发现在自回归解码场景下，量化误差会随时间步累积，主要原因是错误的 Token 尺度。为此，我们提出 KVarN——一种免校准的 KV-Cache 量化器，它对 K 和 V 矩阵的两个轴依次应用 Hadamard 旋转和双尺度方差归一化。我们发现这种组合能够修复异常的 Token 尺度误差，并显著降低相比现有基线的误差累积。在 2-bit 精度下，KVarN 在 MATH500、AIME24 和 HumanEval 等生成式基准测试中，为 KV-Cache 量化树立了新的 SOTA。KVarN 方法的 vLLM 实现已开源：https://github.com/huawei-csl/KVarN

## K-BrowseComp: A Web Browsing Agent Benchmark Grounded in Korean Contexts

## Mellum2 Technical Report

[Mellum2 技术报告](https://arxiv.org/abs/2605.31268)

Mellum 2 是一个开放权重的 12B 参数混合专家语言模型，每个 Token 激活 2.5B 参数，专精于软件工程任务，并在消费级 GPU 上优化推理效率。由 Qwen/Qwen2.5-Coder-32B-Instruct 生成。我们提出 Mellum 2，一个开放权重的 12B 参数混合专家（MoE）语言模型，每个 Token 激活 2.5B 参数。Mellum 2 是一个通用语言模型，专精于软件工程，涵盖代码生成与编辑、调试、多步推理、工具使用与函数调用、智能体编码以及对话式编程辅助，它是专注于补全的 4B 稠密 Mellum 模型的继任者。架构基于混合专家（64 个专家，8 个激活），结合了分组查询注意力（4 个 KV 头）、每四层中三层使用滑动窗口注意力，以及一个单一的多 Token 预测头，该头同时作为辅助预训练目标和内置的推测解码草稿模型；每个选择都通过消融实验验证，并以消费级 GPU 上的推理效率作为设计约束。预训练涵盖约 10.6 万亿 Token，采用三阶段课程，逐步将混合数据从多样化网络数据转向精选代码和数学内容，使用 Muon 优化器在 FP8 混合精度下优化，并采用 Warmup-Hold-Decay 调度（线性衰减至零）。预训练基础模型通过层选择性 YaRN 扩展到 128K 上下文窗口，然后分两个阶段进行后训练（监督微调后接 RLVR），产生两个发布变体：直接回答的 Instruct 模型和在最终答案前输出显式推理轨迹的 Thinking 模型。在代码生成、数学与推理、工具使用、知识和安全基准测试中，Mellum 2 与 4B-14B 范围内的开放权重基线模型具有竞争力，同时每个 Token 的计算量相当于 2.5B 稠密模型。我们在 Apache 2.0 许可下发布基础、指令和思考检查点，以及关于架构决策、数据管道和训练配方的本报告。

## From Activation to Causality: Discovery of Causal Visual Representations in the Human Brain

[从激活到因果：发现人脑中的因果视觉表征](https://arxiv.org/abs/2605.23895)

BrainCause 框架结合生成模型与脑模型，通过因果测试识别有效的神经表征，证明仅凭激活不足以确认概念表征。识别人脑中哪些脑区表征某个视觉概念是神经科学的核心挑战。现有方法通过激活最大化定位粗略的功能区域（如面孔、地点），找出对目标概念激活显著强于其他概念的脑区。然而，仅凭强激活并不能证明该区域表征了概念本身，因为响应可能由相关的视觉或语义线索驱动。我们提出 BrainCause，一个自动化框架，结合生成模型与脑模型来合成受控刺激，并通过定向因果测试验证神经表征。给定一个指定感兴趣概念的查询，我们的框架构建定向刺激集，包括概念图像、移除目标概念但保留其他图像内容的反事实编辑图像，以及包含候选相关干扰物的图像。然后使用图像到 fMRI 编码模型预测脑响应，并搜索对目标概念响应特异性高于相关替代物的表征。BrainCause 返回经过验证的候选表征，并提出后续 fMRI 实验以进一步测试或扩展其发现。我们的方法成功恢复了已知的功能定位，并在数十个概念上识别出新的候选表征，在预测和实测 fMRI 数据上均得到验证。关键的是，我们证明如果没有因果验证，大部分定位将是假阳性，确认了仅凭激活不足以作为表征的证据。

## Where Do Deep-Research Agents Go Wrong? Span-Level Error Localization in Agent Trajectories

[深度研究智能体错在哪里？智能体轨迹中的跨度级错误定位](https://arxiv.org/abs/2606.02060)

深度研究智能体可通过一种以声明为中心的审计框架进行审查，该框架能识别其推理轨迹中的错误跨度，从而超越仅评估最终答案的方式，提升可靠性评估。深度研究智能体通过搜索、工具使用、证据检查和答案合成等长轨迹来完成任务。基于最终答案的评估能判断智能体是否成功，但无法揭示轨迹中哪些部分导致答案不可靠。我们研究了深度研究智能体的跨度级错误定位问题。我们从两个智能体框架、三个骨干模型和三个基准测试中收集了2,790条真实轨迹，将原始日志转换为语义跨度，并通过大语言模型辅助的专家审查标注了有害错误跨度。基于这些标注，我们构建了TELBench——一个包含1,000个实例的基准测试，用于在正常探索、失败搜索、暂定假设和无害噪声中识别错误跨度。我们进一步提出了DRIFT，一种以声明为中心的审计框架，该框架追踪智能体的声明，检查其在轨迹证据中的支持情况，并标记那些无支持或冲突声明影响答案路径的跨度。跨模型家族和审计框架的实验表明，DRIFT将跨度级错误定位和首次错误准确率提升了最多30个百分点。我们的工作为深度研究智能体的可靠性提供了过程级视角。

## Harness-1: Reinforcement Learning for Search Agents with State-Externalizing Harnesses

[Harness-1：利用状态外化框架的强化学习搜索智能体](https://arxiv.org/abs/2606.02373)

一个基于有状态搜索框架、通过强化学习训练的 20B 搜索智能体，通过将语义决策与环境记账分离，在多个领域展现出卓越的检索性能。搜索智能体通常被训练为基于不断增长的对话记录的策略：模型必须决定如何搜索，同时记住已看到的内容、哪些证据有用、哪些约束仍未解决、哪些声明已被核实。我们认为这种设定将过多常规状态管理置于策略内部：强化学习被迫同时优化语义搜索决策和可由环境更可靠维护的可恢复记账。我们提出 Harness-1，一个在有状态搜索框架内通过强化学习训练的 20B 搜索智能体（检索子智能体）。该框架维护环境侧工作记忆，包括候选池、重要性标记精选集、紧凑证据链接、验证记录、压缩去重观察结果以及预算感知上下文渲染。策略保留语义决策：搜索什么、保留或丢弃哪些文档、验证什么以及何时停止。在涵盖网页、金融、专利和多跳问答的八个检索基准测试中，Harness-1 实现了 0.730 的平均精选召回率，比次强开源搜索子智能体高出 11.4 个百分点，并与规模更大的前沿模型搜索器保持竞争力。其在保留迁移基准上的增益尤为显著，表明基于显式搜索状态的强化学习可以产生超越训练领域的检索行为。我们的代码开源在 https://github.com/pat-jj/harness-1。

## ArcANE: Do Role-Playing Language Agents Stay in Character at the Right Time?

[ArcANE：角色扮演语言智能体能否在恰当时间保持角色一致性？](https://arxiv.org/abs/2606.05553)

角色扮演语言智能体需要随叙事推进而动态发展角色，这要求基准测试评估心理轨迹对齐而非静态事实回忆。ArcANE 在将角色弧信息注入模型后展现出更优性能。角色扮演语言智能体应扮演价值观和行为随故事发展而演变的角色，而非维持固定人设。现有基准测试仅衡量特定章节的事实回忆，无法判断回复是否与角色心理轨迹对齐，尤其在源文本未探索的场景中。我们提出 ArcANE（弧感知叙事评估），这是一个自动构建的基准测试，涵盖 17 部小说和 80 个主要角色。角色弧沿心理轴将叙事划分为多个阶段，每个探测问题在不同阶段呈现相同场景，覆盖源文本内外的情境。在六个模型和六种上下文模式下，基于角色弧的条件设置在每种模型上均优于其他所有上下文策略，且在源文本外场景中差距最大——此时检索无法提供任何信息。我们进一步在相同数据上微调开放权重模型，获得 ArcANE-8B/32B，在源文本外场景中进一步扩大了角色弧的优势。
